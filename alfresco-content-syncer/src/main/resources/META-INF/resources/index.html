Quarkus UI | local binaries to Alfresco

Alfresco Content Syncer

Verify the Alfresco connection, choose a site, browse only the folders under its documentLibrary, then sync a local folder and generate a readable CSV report.

Operational Notes

Use dry run first. Successfully transferred files are tracked locally in embedded H2 and skipped on later runs when the local checksum is unchanged. Use Force new version sync to upload the same local file again as a new Alfresco version when a file with the same name already exists in the same remote folder. If the report path is left empty, the server writes a CSV report in the current working directory with a generated filename.

Connection

Validation rules

The Alfresco URL must end with /alfresco. Provide username/password or a ticket, then use Verify connection before browsing sites.
