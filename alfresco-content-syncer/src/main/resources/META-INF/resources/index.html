Alfresco Content Syncer

Operational UI to browse Alfresco folders, launch local folder synchronization jobs, track status, and inspect readable reports. Sync runs on the same host where this application is started.

Operational Notes

Use dry run first. Remote delete is disabled by default because it is destructive. Existing files are updated using size, timestamps, and a persisted SHA-256 manifest to avoid false updates when only local timestamps drift.

Connection

Tip

Use username/password or a ticket. If syncer.ui.auth-token is configured, the same token protects all /api/* endpoints.
